# members

mcpp社区 成员&开发者 申请入口 -- 增加社区成员归属感, 并开放一些预研/探索性质项目的私有仓库权限, 方便社区成员更早能参与讨论和开发

## 介绍

- [members](https://github.com/orgs/mcpp-community/teams/members) : 所有参与过或有意愿参与社区讨论和活动的成员, 面向所有人
- [vteam](https://github.com/orgs/mcpp-community/teams/vteam) : 设置志愿者团队, 默认有组织所有公开仓库的triage权限, 以及部分仓库写权限
- [coreteam](https://github.com/orgs/mcpp-community/teams/coreteam) : 负责推动社区发展, 以及把抽象问题和目标拆解为可执行的任务

## 具体步骤

### 加入社区

1. 在本仓库创建一个 [Join Members Issue](../../issues/new?template=join-members.yml)
2. 提交即可，系统会自动获取你的 GitHub 用户名
3. 将自动发送邀请，无需审核

### 加入 VTeam（志愿者团队）

1. 在本仓库创建一个 [Join VTeam Issue](../../issues/new?template=join-vteam.yml)
2. 填写以下信息：
   - **推荐人**（可选）：如果有现有成员推荐，请填写其 GitHub 用户名
   - **申请理由**：说明为什么想加入志愿者团队
   - **过往在社区中的贡献记录**：列举你在 mcpp 社区或相关开源项目中的贡献
3. 提交即可，系统会自动获取你的 GitHub 用户名
4. 等待审核者的批准

### 审核流程

VTeam 申请需要以下审核：
- 需要 **@sunrisepeak** 的审批
- 需要至少 **一位 coreteam 成员** 的审批

审核者在 issue 评论中回复：
- `/approve` - 批准申请

当所有审核条件满足后，系统将自动处理邀请。

## 注意事项

- issue 作者即为申请者
- 推荐人信息有助于加快审核流程
- 如有疑问，可在 issue 中提问或联系维护者
